# Seven Star Systems Specification

Names are locked: Quiet Ambition, Rebuild, Study, Money, Body, Connection, Creation.

## Shared domain model

Each owner System needs a definition, diagnostic answers, checklist/action definitions, persisted actions, progress inputs, Glow contribution, goals/Plans, blockers, latest Timeline event, advice, next move, and future prediction. Progress stores components and explanation, not a random percentage.

## System meanings and initial action focus

| System | Meaning | First evidence-backed action |
|---|---|---|
| Quiet Ambition | private direction, identity, discipline | define one private goal and 20-minute move |
| Rebuild | repair damaged/neglected structure | name repair area and first stable brick |
| Study | deliberate learning/mastery | set target, proof, and one study block |
| Money | financial reality and earning | classify pressure and complete one stabilizing action |
| Body | sleep, energy, pain, food, movement, care | energy/load check and feasible body action |
| Connection | people, support, repair, boundaries | name open loop and next conversation |
| Creation | product/art/code/deliverables | define smallest shippable output |

## Progress

Progress combines completed/action ratio, goal progress, seven-day consistency, outcome-return rate, and verified System Glow; unresolved blockers and missed returns reduce it. The API returns every component and source count.

## Integration

System diagnostics/actions emit Timeline events, update Today Body/Mind/Life and Map nodes, create eligible Glow, and can open Goal/Plan/Research/Community hooks. No System becomes a generic tab detached from evidence.
